Project Summary

Design, % 40
Components, % 40
Data Visualisation, % 20
Following a series of constantly evolving requirements, pushing our teams into different research areas and tasks, the final deliverable for Team 37, as agreed with the client, will be the completed version of PEACH Design Guide website, which has been started back in Term 1.
The website will be used as a means of unifying the design elements as well as components and their code across various platform. Any team-member from the same project can access and review the guidelines about proper integration and usage of the already predefined elements. They can easily and seamlessly updated or changed using our documentation, allowing the change to spread across all other teams, keeping everyone informed and aware.

Key Features

  • Smart Sidebar
  • Live Editor
  • Design
  • Components
  • Data Visualisation (D3)

The smart side bar not only act as breadcrumbs but also as easy navigation through the pages.

Smart Side Bar
The smart side bar follows the user and highlights the current section as the user scrolls through the long sections of design, components or data visualisation. This is analogous to a real time breadcrumb so the user does not get lost in the sea of content.
The smart side bar headings are also hyperlinked to allow navigation to other sections.

Beneath each component example, code is displayed within a live code editor.

Peach Live Editor
The live code editor means developers are able to edit the component examples and see the results of their changes in real time.
Developers can modify the component to suit their need within the Design site, then simply copy the code across to their project.

Within the Design page, there is a comprehensive collection of design guidelines ranging from layout to colour styles.

Peach Design
These design guidelines are aimed to help future peach developers to create a unified UI. The Design page contains guidelines of the layout, fonting, coloring and much much more information about how the website is recommended to look and feel, providing examples of subtle cases when bad design choices might lead to incapability to support various screen resolutions, languages etc.
The pictures under each of the sections help to illustrate how design decisions should be made when words are insufficient to convey the idea.

Within the Components page, each PEACH component is listed, with an embedded example.

Peach Components
All components available for use within the PEACH project are listed, with embedded examples, that the user is able to interact with (where possible).
At present this list contains only default React-Bootstrap components, however the list will be updated once bespoke PEACH components are developed by other development teams.

Interactive examples of the data visualization library D3 are embedded within the D3 tab of the site.

Peach Data Visualisation
Extending the view of the site as a single point of reference for all PEACH development and following new CLIENT requirements, it was decided that part of the site would act as a showcase to the visualization capabilities and aesthetic qualities of D3.
Beneath each example is the accompanying code. Developers can therefore quickly copy across the examples to their project, and modify the visualizations according to their needs.
