Architecture

1. Categorisation

Our design guide is categorised into three sections, namely, design, components and data visualisation. These sections form the bulk of what developers will need to develop Peach.

2. Ease of Use

We laid out the design guide in the format because it's natural for a developer to be looking for one particular type of design element for a feature. The user may have three tabs open at the same time to access all areas if desired. Categorising design philosophies allows for lesser clicks and movement between pages.

3. Intuition

Another reason we chose this layout is also because it is very intuitive to understand. Any developer can easily access the design guide and find where things are due to its intuitive layout.

Design Patterns

Flyweight

Flyweight aims to make web pages where there are large amounts of resources such as images more efficient. This is especially useful for the design guide since a large amount of images are loaded. The Design and Component sections are separate and long pages with lots of contents to load. If the user switches regularly between the two, resources will be wasted to reload these long pages. To optimise the viewing experience, a 'Factory' is created to cache and reuse existing instances.

Separation

A common design pattern in React development is the separation of components into container and presentational components. This allows components to be reused much more easily. In the components page, all the functionality and logic of the embedded component examples was contained within individual container component classes. Separate from this, for the specific examples on the page, each component has a presentational component class, which describes how the example component will look.

Private Class Data

In object oriented programming, there is a need to encapsulate class data after initialization, to prevent data manipulation when it is no longer desirable. The use of React components in site development prevents this. Whenever a component is defined, taking in the ‘props’ object as an input, this props object is read only, thereby preventing the component from modifying the object itself.

Adapter

The adapter design pattern allows a high level interface that grants access to subsystems for the design guide. This is implemented through our top menu bar which acts as navigation to design, components and data visualisation pages.

Prototype

The components section makes heavy use of prototypes throughout in various forms. For example with regard to CSS, many components were instantiated with a certain ‘theme’ of CSS attributes, which were then modified later on.

Singleton

A singleton pattern creates a class, for which three can only exist one instance. Throughout all of the example components, whenever component object is exported, they are exported as a singleton. Every other file that imports the file (component file) will get the same object.

Development Tools

Advanced Text Editors

Speed

Due to the lightweight nature of text editors, they offer a very fast and smooth usage experience. Rarely will the user experience freezes or ‘lags’.

Familiarity

Text editors usually only natively support the core functions that one would expect from a traditional editor. These functions, when combined with shortcuts allow the user to become a respectable level of proficiency rapidly.

Customisation & improvements

Some text editors allow extensions to be installed to mimic the functions of their more powerful IDE counterparts. For instance, Google libraries for emacs. Some more modern text editors such as Atom provide project management options as well as customisation of a list of the languages recognised.

Consistent style

In many scenarios, a clear structure and declaration of style allows ease of understanding of code and thus negates some of the use of of IDEs. In particular in the sense of understandability and readability.

Stability

Most text editors do not often have large updates or patches that completely overhauls the software. This allows the user to be focus on the software content rather than the tool being used.


Some popular text editor tools are Atom, Sublime and Notepad++. In case of web development, a web browser is usually used in conjunction to provide compilation and execution. Many modern browsers today such as Google Chrome and Mozilla Firefox also offer debugging consoles. In our case, we also use our command console of choice along with npm to build and run our react application.

Integrated Developer Environments (IDEs)

Integrated environment

Web IDEs provide an integrated environment for developers. This means that it contains an almost comprehensive array of tools that the developer may use throughout the development of the project. Some of these tools include but are not limited to the following: debugger, navigation, code auto completion and class explorers.

Variety

This variety of tools provide the integrated environment which is the main advantage IDEs have against lightweight text editors.

Speed

Although IDEs will always provide more tools than simple text editors, these tools are not usually required or used by the developer. Thus these tools only make for a slower use experience.


An example of a popular web IDE is webstorm. This is an extremely popular IDE due to its reputation for stability. Webstorm is excellent for our project since it has npm built in so one does not have to alternate between a command console and browser to work on the project.
