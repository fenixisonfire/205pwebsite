Platform, OS and Environment

  • Platform
  • OS
  • Environment
Our project is mainly concerned with launching the Design Guide website. For our testing, we used localhost, specifically “localhost:4000”, which is the default port for a npm react web-application. Additionally, the application can be run using services similar to Azure by Microsoft or AWS by Amazon. However, the project cannot be deployed to GitHub pages, as it requires server-side compilation. The website does not require a database.
As the application is web-based, the execution of it runs only in browsers. As we have tested, all of the most commonly used browsers such as Internet Explorer, Google Chrome, Mozilla Firefox and Safari support our application. And as the browsers are added to almost any operating system (OS) by default, we can say that the project can be used under any OS. Moreover, all contents within the website such as Data Visualisation (D3) and Components are all optimised for different platforms.
As previously mentioned, the execution of our application locally or on a server can be done using Azure services, AWS or other similar products, but not GitHub Pages. This is due to the fact that the project requires a set of commands to be run prior to opening the link. There are 3 commands to install all the packages, and 1 command which should be run everytime the application is shut down.

Languages, Libraries and APIs

Libraries

D3
D3.js was written by Mike Bostock while working at Stanford Visualisation Group. It stands for Data Driven Documents. Essentially, D3 allows for easy integration of dynamic and interactive data visualisation. It achieves this by attaching data to Document Object Model (DOM) elements, then CSS3, HTML and or SVG can be used to showcase the data. Interactivity can be implemented through D3.js data-driven transformations and transitions.[d1]

Key Features
  • Binds data to Document Object Model
  • Allows CSS3, HTML and or SVG to showcase data
  • Interactivity through D3 data-driven transformations
  • Dynamic data visualisation
  • Reusable JavaScript code allows for custom functions


Usage
Our client and professional advisors recommended D3 to us for creating the front end data visualisation. Other teams on the Peach project will generate the requested data on the backend and feed these data to the front end where we will use D3 to represent parts of the data. These data will generally include patient information and medical history where diagrams and graphs can be greatly beneficial for doctors to make quick and accurate diagnostics.

Advantages
  • Open Source
  • Many powerful features which are missing from its competitors. Eg:
    • Transformations and transitions
    • “Enter and Exit”
    • Similar syntax to jQuery[d2]
    • Used by trustworthy sites such as NYTimes and Weather.com


Disadvantages
  • Does not support legacy web browsers such as Internet Explorer 8 (IE8)
  • May require additional plugins such as “aight plugin” for manual support

React Playground
React Component Playground is a React component developed by Formidable, a Seattle based consultancy. It renders editable source code and a live preview of other React components. It essentially allows for the implementation of live code editors. [rp1]

Key Features
  • Allows editing of source code to illustrate changes
  • Live preview of React components

Usage
For the components page of the Design Guide, we will be able to use the functionality provided by React Component Playground to have live code editors embedded within the Components list, alongside each component example. This way, site users can edit source code on the fly and see the result rendered in real time. There is also the potential to use it within the D3 tab to embed editable charts.

Advantages
  • OpenSource
  • Most widely used code editor for React sites
  • Well maintained by Formidable

Disadvantages There remain a number of unresolved issues with the component, listed here. [rp2]

Alternatives
Upon research, there would appear to be little or no alternative than to use React-Component-Library to render a live preview of editable components and source code, however there are other React based playgrounds such as JSFiddle and RNPlay.
React-D3
React D3 is a plugin for the D3 library, built by LiJung Chi, that allows the use of React methods within the D3 API . The library is a collection of React components that can be used to create custom D3 charts in a modular fashion. The library also comes with a variety of D3 template charts pre-converted to React components, for those unfamiliar with D3 [rd1].

Key Features
  • Many template charts, allows users unfamiliar with D3 to quickly implement visualisations.
  • Reusable modular React component library to create custom D3 charts.
  • Supports high level components which can be used to install and create a chart within a component Utilizes power of D3 to automate scaling ranges, normalize data and construct legends.

Usage
The library will be used to embed examples of D3 visualisations within the visualisations tab of the Design Guide. The examples will also have the accompanying React D3 code we used to implement them. As the majority of future PEACH development is to be done in React, we encourage the use of this library amongst all data visualisation teams. Advantages

Advantages
  • OpenSource
  • Able to build custom D3 visualisations entirely using modular React components
  • Extensive list of template charts

Disadvantages
  • Modularity of components library means some customizations not possible[rd2]
  • Library not maintained
  • Not widely used

    Bootstrap
    Bootstrap was developed by Mark Otto and Jacob Thornton as a framework to encourage consistency across internal tools at Twitter. [b1] It has since evolved into a widely used front-end framework for designing websites and web-apps. Having defined the new phrase - “bootstrap it” in the web development world, bootstrap not only offers automatic scaling across multiple platforms but also a plethora of components at the user’s disposal.

    Key Features
    • Features CSS preprocessors, Less and Sass within its source code
    • Built in vanilla CSS
    • Automatically scales the product website to custom window sizes for different devices
    • Custom HTML and CSS components
    • Extensive documentation
    • jQuery plugins
    [b2]

    Usage

This is intuitive since it is a standardised platform where styles and components such as layout, panels and buttons are ready to be plugged in. This allows for a fast setup and maintenance which enables us to focus more on the contents of the design guide. It will also be inherently compatible across all platforms and devices.

    Summary of Decision

    React

    The decision to use React to build the actual Design Guide came relatively late in the project. Having previously tried with various degrees of success to implement the components page purely using HTML and CSS, we realized that the system would run more smoothly if we built the site using a React framework. React is highly suited to handle the frequent state changes invoked by the components page.

    React Playground

    There are a number of playground libraries built for React, however only React Components Playground offered the ability to live edit embedded React components. It efficiently uses React’s lightweight virtual DOM rendering system to handle live code editors. In our page, the result after editing an example component is instantaneous.

    React D3

    Given that D3 is the gold standard for data visualization, we needed to integrate it within our React site. However this is a notoriously tricky task. The React D3 library broke this problem down by offering React components which can be used to construct D3 graphs. This greatly smoothed the process of embedding D3 examples in our site. The library also came with a number of prebuilt template charts, which is highly useful for developers who are unfamiliar with D3.

    D3

    D3.js is to be used as it is it is the most customizable of any of the libraries researched, whilst it also allows for dynamic interactive data displays. It would have been preferential to be able to use N3 charts alongside D3 to allow further functionality, however given that all Peach teams have reached the unanimous decision to develop with React, rather than Angular, this is not a possibility.
